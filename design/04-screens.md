## 4.2 献立カレンダー（トップ）

### 画面構成

上下2段のカレンダー構成。

**上段: 週カレンダー（当日起点 or 起点ずらし）**

- 7 日分のセルを横並び
- 左右に `◀` `▶` ボタン。クリックごとに 1 日ずつ前後に起点をずらす
- 初回表示時の起点は「当日」（左端が当日）
- 月カレンダーの月送りとは独立（週カレンダーは自身の状態を保持）
- 各セルの表示内容は月カレンダーの同日セルと同じ（料理名のリスト）
- D&D のドロップ先・ドラッグ元の両方になる

**下段: 月カレンダー（当月 or 任意月）**

- 週の開始は **日曜**
- 7列 × 6週の一般的なグリッド
- ヘッダ: `◀ 2026年4月 ▶`
- 各セル: 日付番号 + 当日の料理名リスト
- 当日セルは背景色などで強調
- D&D のドロップ先・ドラッグ元の両方になる

### セル内表示ルール

- 日付番号（小さめ、セル左上）
- その日の `Dish.name` を上から順にリスト表示
- セル高さを超える場合は **セル内スクロール**（縦スクロール）
- セル全体がクリック可能（`date` を持って献立入力モーダルを開く）
- セル全体が D&D のソース/ターゲット

### 当日セルの強調

- 背景色をアクセントカラーで控えめに塗る
- 日付番号を太字 + アクセントカラーにする
- 月カレンダー側のみ強調（週カレンダーは常に当日を含むので強調不要）

### ドラッグ&ドロップ仕様

- **ドラッグ単位**: 日単位（その日の全料理をまとめて）
- **ドラッグ元**: 月カレンダーのセル、週カレンダーのセル
- **ドロップ先**: 月カレンダーのセル、週カレンダーのセル
- **対応方向**: 月↔月、週↔週、月↔週（全方向）
- **視覚フィードバック**:
  - ドラッグ中のセル: 半透明化
  - ドロップ可能なセル: ホバー時に枠線 or 背景色変化
- **ドロップ時の挙動**:
  - 同じ日付へのドロップ: 何もしない
  - 空のセルへのドロップ: そのままコピー（確認不要）
  - 既に料理があるセルへのドロップ: **確認ダイアログ**を出す
    - 選択肢: `上書き` / `入れ替え` / `キャンセル`
    - 上書き: ドラッグ元はそのまま、ドロップ先を新内容に置換（コピー）
    - 入れ替え: ドラッグ元とドロップ先の内容を交換（スワップ）
    - キャンセル: 何もしない

### データの読み込み範囲

- 月カレンダー: 表示中の月の前後1日を含めた期間 / 週の欠けを考慮し、グリッドで表示する全 42 日分
- 週カレンダー: 表示中の 7 日分
- 画面表示時に `listMenusInRange` で取得して保持

## 4.3 献立入力モーダル

### 開くタイミング

- 月カレンダー / 週カレンダーのセルをクリックしたとき
- 対象は「クリックした日付」のみ

### モーダル構成

```
┌─ 2026年4月17日(金) の献立 ──────────────────[×]┐
│                                                  │
│  ＋料理を追加  [レシピから選択 ▼] [直接入力]    │
│                                                  │
│  ┌─ 料理1 ────────────────────[↑][↓][🗑]─┐    │
│  │ 料理名: [ 焼き鮭                     ]  │    │
│  │ レシピ参照: 焼き鮭(基本)  [解除]         │    │
│  │ 材料:                                  │    │
│  │   [鮭     ][1    ][切れ  ][🗑]         │    │
│  │   [塩     ][少々  ][      ][🗑]        │    │
│  │   ＋材料を追加                          │    │
│  │ メモ: [グリルで両面5分               ]  │    │
│  │ [レシピを更新]  ← 差分ありの時のみ表示  │    │
│  └────────────────────────────────────┘    │
│                                                  │
│  ┌─ 料理2 ────────────────────[↑][↓][🗑]─┐    │
│  │ 料理名: [ 漬物                       ]  │    │
│  │ (直接入力)                              │    │
│  │ 材料: （空でもOK）                       │    │
│  │ [新規レシピとして保存]  ← recipeId=null時のみ│
│  └────────────────────────────────────┘    │
│                                                  │
│            [キャンセル]  [保存]                  │
└──────────────────────────────────────────────┘
```

### 仕様詳細

#### 料理追加方法

「＋料理を追加」の右に 2 種類のボタンがある。

1. **レシピから選択**: `RecipePicker` コンポーネント（検索ボックス付きリスト）をポップオーバーで開く
   - 検索: `Recipe.name` に対する部分一致（大文字小文字無視）
   - 候補をクリックすると `Dish` として追加
   - 追加時、`Recipe.ingredients` を複製して `Dish.ingredients` に入れる
2. **直接入力**: 空の料理カードを追加（`recipeId: null`、名前・材料・メモすべて空）

#### 料理カード

- ヘッダ右端: 並び替えボタン（↑↓）、削除ボタン（🗑）
- 料理名: 常に編集可能
- レシピ参照表示:
  - `recipeId !== null` の場合: 「レシピ参照: (レシピ名)」と表示、`[解除]` でリンクを外す
  - `recipeId === null` の場合: 「(直接入力)」と表示
- 材料リスト: 各行 `[名前][分量][単位][🗑]` + 「＋材料を追加」
- メモ: テキストエリア 1 行

#### レシピ更新ボタン

- 表示条件: `recipeId !== null` かつ、`Dish.name / ingredients / procedureMemo` が元 `Recipe` と差分あり
- 押下時:
  1. 確認ダイアログ「レシピ『(レシピ名)』を更新しますか？」→ OK / キャンセル
  2. OK の場合、`Dish` の内容で `Recipe` を `updateRecipe` する

#### 新規レシピ保存ボタン

- 表示条件: `recipeId === null` かつ、`Dish.name` が非空
- 押下時:
  1. 確認ダイアログ「この料理をレシピとして保存しますか？」→ OK / キャンセル
  2. OK の場合、`Dish.name / ingredients / procedureMemo` を新規 `Recipe` として `createRecipe`
  3. `Dish.recipeId` を新規レシピの id に更新

#### 並び替え

- ↑↓ ボタンで料理カードを入れ替える
- 先頭/末尾の料理は該当ボタン無効化

#### 保存 / キャンセル

- `[保存]`: `saveMenu` を呼び、`MenuEntry.dishes` を更新。モーダルを閉じてカレンダーを再描画
- `[キャンセル]`: 変更を破棄して閉じる（変更ありの場合は確認ダイアログ）
- 全料理を削除して保存した場合は、その日の `MenuEntry` ごと DB から削除

## 4.5 買い物リスト画面

### レイアウト

```
┌──────────────────────────────────────────────┐
│  買い物リスト                                  │
│  期間: [2026/04/17] 〜 [2026/04/23]  [生成]   │
│  並び順: ( ) 日付順  ( ) 材料名順              │
├──────────────────────────────────────────────┤
│  ☐ 鮭         1切れ      (4/17 焼き鮭)       │
│  ☐ 塩         少々        (4/17 焼き鮭)       │
│  ☐ 豆腐       1/2丁      (4/17 豆腐の味噌汁)  │
│  ☐ わかめ     ひとつまみ  (4/17 豆腐の味噌汁)  │
│  ☐ 鶏もも肉   300g       (4/18 親子丼)        │
│  ...                                           │
│                                                │
│  [すべてチェック解除]                          │
└──────────────────────────────────────────────┘
```

### 仕様

#### 期間指定

- 開始日・終了日の 2 つの日付入力
- デフォルト: 開始日 = 当日、終了日 = 当日 + 6 日（当日から1週間）
- `[生成]` ボタンで下の一覧を再計算

#### 並び順

- ラジオボタン切替
  - **日付順**（デフォルト）: `date ASC`、同日内は `Dish` の順序、さらに `Dish` 内は材料の順序
  - **材料名順**: 材料名の文字列昇順
- チェック済みアイテムは並び順内でその位置に留まる（末尾移動はしない）

#### 一覧表示

- 1 行 = 1 材料（集計・合算は一切行わない）
- 表示項目: `☐ / 材料名 / 分量 + 単位 / (日付 料理名)`
- 「日付 料理名」は出典表示
- **空の材料（`name`空 or `amount`空）**: 出力しない（料理名のみの記録や未入力行を除外）

#### チェック状態

- 画面内の一時状態のみ（ページ離脱・再訪で消える）
- チェック済みは **取り消し線** のみ（並び順は変化しない）
- `[すべてチェック解除]` ボタンで一括解除

#### データ取得

```typescript
const menus = await listMenusInRange(from, to)
// 各 MenuEntry.dishes.forEach(dish => dish.ingredients.forEach(ingredient => ...))
// の全展開で 1 行を作る
```

## 4.6 設定画面

### レイアウト

```
┌──────────────────────────────────────────────┐
│  設定                                          │
├──────────────────────────────────────────────┤
│ ■ データ管理                                   │
│   [JSONエクスポート]  [JSONインポート]         │
│   最終エクスポート: 2026/04/15 21:30           │
│   ⚠ 30日以上エクスポートされていません         │
│                                                │
│ ■ ストレージ                                   │
│   永続化ストレージ: 許可済み ✓                 │
│   [永続化を要求]  (未許可時のみ活性)           │
│   使用容量: 2.3 MB / 推定上限 約500MB          │
└──────────────────────────────────────────────┘
```

### 仕様

#### JSONエクスポート

- クリックで全データを集約した `BackupFile` JSON をダウンロード
- ファイル名: `cookpath-backup-{yyyyMMdd-HHmm}.json`
- 完了後、`meta.lastExportedAt` を現在時刻に更新

#### JSONインポート

- `<input type="file" accept=".json">` でファイル選択
- 選択後、**必ず確認ダイアログ**:
  > 現在のデータはすべて削除され、インポートしたデータで置き換えられます。続行しますか？
- OK の場合:
  1. `formatVersion` 検証。不正ならエラー表示して中断
  2. `recipes` / `menus` テーブルを全削除
  3. ファイル内容を `bulkPut`
- `meta.lastExportedAt` はインポートで変更しない（直前のexport日時は書き換えない）

#### 最終エクスポート日時

- `meta.lastExportedAt` を日本語表記で表示（`null` なら「未エクスポート」）
- 30 日経過 or `null` の場合に警告バナー（⚠ アイコン + 赤系文字）

#### 永続化ストレージ

- 現在の状態（`isPersisted()`）を表示
- `[永続化を要求]` ボタン（`isPersisted() === false` のときのみ活性）でリクエスト
- 使用容量は `estimate()` の結果を MB 単位で表示

## 4.8 起動時の処理

`App.vue` の `onMounted` 等で以下を実行。

1. `isPersisted()` を確認。false なら `requestPersistence()` を一度だけ試行（結果は握りつぶす）
2. `getLastExportedAt()` を取得し、ヘッダやトップに警告を出すかどうかの判定に使う（設定画面以外でも警告を出したい場合は要検討、初期は設定画面のみでよい）
