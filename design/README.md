## プロジェクト要旨

- **目的**: 個人（1人）の日々の献立計画と、その献立に必要な食材の買い物チェックリスト作成を効率化する
- **対象環境**: PC（一般的なモニタサイズ）。スマホ対応は行わない
- **配信**: GitHub Pages（フロントエンドのみ、サーバー無し）
- **データ保存**: IndexedDB（Dexie.js）+ Persistent Storage API、JSON export/import でバックアップ
- **言語**: コードの識別子は英語、コメント・PR・コミットメッセージ・UI文言は日本語

## 重要な前提・非目標

- ネットスーパーとのAPI連携は行わない（買い物チェックリスト出力までが責務）
- スマホ・タブレット最適化は行わない
- マルチユーザー / アカウント機能は無し（ブラウザに紐づく単一ユーザー前提）
- 朝・昼・夕の区別は持たない（1日 = 1食、ただし1食内に複数料理を持てる）
